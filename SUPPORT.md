# Support

## Getting Help

Before opening an issue, check these resources in order:

1. **README** — [README.md](README.md) covers installation, daily workflow, and all helper scripts.
2. **GitHub Issues** — Search [existing issues](https://github.com/DiNaSoR/Mnemo/issues) to see if your question is already answered.
3. **GitHub Discussions** — For questions, ideas, and general conversation: [Discussions tab](https://github.com/DiNaSoR/Mnemo/discussions).

## Bug Reports

Use the [Bug report](.github/ISSUE_TEMPLATE/bug_report.yml) template and include:
- Mnemo version (check `VERSION` file at repo root)
- OS and shell version
- Full console output

## Response Time Expectations

| Channel | Expected response |
|---------|-------------------|
| GitHub Issues (bugs) | 7 days |
| GitHub Discussions | Best effort |
| Pull Requests | 7–14 days for initial review |

This is a community-maintained project. Response times are best-effort and may be longer during holidays or high-demand periods.

## Feature Requests

Open a [Feature request](.github/ISSUE_TEMPLATE/feature_request.yml) issue. Features with a clear use case and a proposed implementation will be prioritized.

## Scope of Support

Mnemo supports the installers and helper scripts in this repository. Issues specific to:
- Cursor IDE behavior → report to [Cursor](https://cursor.com)
- OpenAI / Gemini API → report to the respective provider
- `sqlite-vec` or `mcp[cli]` packages → report to those projects upstream
